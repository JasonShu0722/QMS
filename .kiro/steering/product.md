# 一、项目简介

## 1.1 产品名称及用途：
质量管理系统，建立覆盖质量各板块管控的信息化管理系统，同步衔接现有的信息系统，实现各版块质量数据智能化统计以及跟进。

## 1.2 主要特性和功能：
- 质量问题登录、跟进
- 质量数据（来料、过程、客户、分供方绩效）统计，实时计算推移，趋势预警以及超标报警，自动联动内部IMS系统拉取物料到料以及成品产出数据
- 经验教训归拢罗列，从质量问题延伸沉淀，新品项目进行点检
- 新品试产、初期流动生产数据汇总，指标批次推移跟进，超标问题闭环跟进
- 客户审核、内部过程审核、分层审核（管理层级别）、供应商准入审核、供应商年度审核等各类审核结果以及问题清单闭环跟进
- 索赔管理（含客户索赔、供应商索赔），自动联动内部OA、SAP系统
- 供应商变更提交、跟进
- 通过数据仪表盘、任务看板可视化管理全面的质量数据
- 接入AI实现趋势诊断、问题回复诊断

## 1.3 目标用户或使用场景
- 检验员：数据及问题录入
- 质量工程师：问题跟进、数据统计分析
- 供应商、其他责任板块工程师：质量问题、审核问题应答
- 各部门及公司领导：查阅可视化面板，了解整体质量状况

## 1.4 业务领域及背景
汽车电子制造行业，新能源汽车控制系统设计与加工制造

# 二、功能模块规划

## 2.1 系统权限管理

### 2.1.1 核心架构：
基于“功能-操作”的拆解细分式权限控制体系，由管理员进行权限分配而非简单的固定角色。
- 权限配置：
    功能模块维度：细分至一级/二级/三级菜单（如：供应商管理 > 绩效考核 > 月度评分）。
    操作权限维度：针对每个功能点，独立配置以下权限的开关
    - 录入/新建：是否能发起单据。
    - 查阅：是否可见（含对应板块的可视化看板，特殊逻辑：供应商账号仅能查看关联到其自身供应商名称的数据）。
    - 修改/编辑：是否能修改未归档数据。
    - 删除：是否能软删除数据（逻辑删除，数据库保留记录但前端不可见）。
    - 导出：是否有权下载 Excel/PDF（防泄密管控）。

### 2.1.2 动态配置UI：
提供一个网格化（Grid/Matrix）的配置界面，管理员通过勾选“复选框”实时调整权限集合，后端无需重启即可生效。

### 2.1.3 用户基本信息配置：
- 公司用户注册时填写：用户名、姓名、电话、邮箱、部门（序列下拉选择）、职位。注册后默认为“待审核”，需管理员在后台审核通过后方可登录。
- 供应商用户注册时填写：用户名、姓名、电话、邮箱、供应商名称（核心校验逻辑：输入“供应商名称”时，前端必须调用 API 模糊搜索系统内的《供应商名录》，用户必须从中匹配一个现存的供应商名称，不允许手动输入不存在的名称。）、职位。注册后默认为“待审核”，需管理员在后台审核通过后方可登录。

### 2.1.4 操作日志：所有关键操作（提交、删除、修改）必须记录后台日志（谁、什么时间、做了什么），管理员可以查阅操作日志，并可以进行筛选查阅。

## 2.2 个人中心（默认首页）

### 2.2.1 页面布局与逻辑
- 设计原则：作为系统登陆后的默认落地页，采用“卡片式”布局，强调“任务驱动”。
- 数据加载：页面加载时，自动触发 API 从各个业务模块（客诉、IQC、审核等）拉取与当前登录用户 ID 关联的待办事项。

### 2.2.2 个人信息模块
- 基础展示：显示用户头像、姓名、部门、职位。
- 账号安全与设置：
    - 修改密码：支持用户自主修改登录密码。
    - 电子签名管理：用户可上传个人的手写签名图片，网站自动实现图片背景透明化。在后续审批 8D 报告或审核单时，系统将调用此图片自动生成电子签章。

### 2.2.3 待办事项
- 聚合逻辑：这是一个“任务聚合器”。系统需遍历所有业务表，筛选出 `Current_Handler_ID` (当前处理人) 等于 `Current_User_ID` 的记录。
- 任务分类与展示：
    - 显示要素：任务类型（如“8D报告审核”）、单据编号、紧急程度（高/中/低）、剩余处理时间（倒计时）。
    - 快捷跳转：点击某条待办，直接跳转到对应模块的具体单据详情页进行处理。
- 状态颜色标识：
    - 🔴 已超期：倒计时 < 0。
    - 🟡 即将超期：倒计时 ≤ 72小时。
    - 🟢 正常：倒计时 > 72小时。

### 2.2.4 站内信通知
- 消息类型：
    - 流程异常通知：如“您提交的索赔单已被驳回”。
    - 系统提醒：如“本月客诉数据已生成，点击查阅”。
    - 预警通知：如“供应商 A 的不良率连续3天上升，请关注”。
- 交互功能：
    - 支持“一键标记为已读”。
    - 未读消息在右上角铃铛图标显示红点数字。

### 2.2.5 公告栏
- 内容分类：系统公告（维护通知）、质量警告、体系文件更新通知。
- 强制阅读机制：
    - 对于被标记为“重要”的质量预警（如：客户发生重大投诉，全员需知晓），用户登录后弹窗提醒点击查阅，系统后台记录“阅读时间”和“阅读人 ID”。
    - 未读的重要公告应置顶并高亮显示。

## 2.3 系统管理与全局配置
*（本模块仅限管理员可见）*

### 2.3.1 全局任务统计与监控
- 用途：用于监控团队整体的工作负载和流程瓶颈，而非具体操作单个任务。
- 统计视图：
    - 待办分布图：按“部门”或“人员”统计当前的待办任务数量（例如：SQE组目前积压了 50 个 8D 报告）。
    - 逾期分析：列出当前所有已逾期（红灯）的单据清单，支持按“逾期时长”排序，方便管理者进行督办。
- 批量转派：当某位工程师离职或请长假时，管理员可在此界面将其名下的所有“待办任务”批量转移给另一位用户。
- AI诊断：调用AI工具诊断评价目前现状

### 2.3.2 消息通知机制配置
- 触发规则引擎：
    - 管理员可自定义业务触发器（Trigger）。
    - *配置逻辑举例*：[选择对象：客诉单] + [选择条件：状态=已驳回] -> [执行动作：发送邮件给提交人]。
- 升级策略：
    - 定义“超时自动升级”规则。
    - *配置逻辑举例*：若单据在 [当前节点] 停留超过 [48小时]，自动触发 [邮件抄送] 给 [当前处理人的上级]。
- 通道配置：
    - 配置 SMTP 邮件服务器参数。
    - 配置企业微信/钉钉/飞书的 Webhook 地址（用于接收系统级预警）。

### 2.3.3 人员信息及权限配置
- 用户管理：
    - 审批注册：查看并处理状态为“待审核”的注册申请（含内部员工及供应商）。
    - 账号维护：重置用户密码、冻结/解冻账号（如供应商暂停合作、人员离职时冻结账号）。
- 权限矩阵配置：
    - 实现 2.1.1 章节定义的“网格化配置界面”。
    - 管理员在此处点击账号名弹出配置界面，勾选每个角色对应的“不同一二三级模块的录入/查阅/修改/删除/导出”权限，保存后实时生效。

## 2.4 质量数据面板

### 2.4.1 数据集成与计算逻辑
- IMS系统对接：
    - 接口定义：开发定时任务，每日凌晨 02:00 从内部 IMS 系统同步“物料入库批次数”、“物料入库不合格批次数”、“物料入库总数量”、“成品出库总量”等分母数据。
    - 自动计算公式：
        - 来料批次合格率 = ((物料入库批次数-物料入库不合格批次数) / 物料入库批次数) * 100%
            注：
            - 按供应商进行区分统计并计算总数
            - 按日、月统计推移并合计年度数据
        - 物料上线不良PPM = (物料上线不良数 / 物料入库总数量) * 1,000,000
            注：
            - 按供应商进行区分统计并计算总数
            - 可实现按物料类型分类统计
            - 按日、月统计推移并合计年度数据
        - 制程不合格率 = (完成制程不合格品数 / 成品产出入库数) * 100%
            注：
            - 按责任类别进行区分统计并计算总数
            - 按日、月统计推移并合计年度数据
        - 0KM不良PPM = (0KM客诉数 / 成品出库总量) * 1000,000
            注：
            - 按产品类型进行区分统计并计算总数
            - 按日、月滚动推移并合计年度数据
        - 售后不良PPM(3mis) = (3mis客诉数 / 3个月滚动出货量) * 1000,000
            注：
            - 按产品类型进行区分统计并计算总数
            - 月度滚动推移
        - 售后不良PPM(12mis) = (12mis客诉数 / 12个月滚动出货量) * 1000,000
            注：
            - 按产品类型进行区分统计并计算总数
            - 月度滚动推移

### 2.4.2 数据可视化展示
- 公司用户视图：
    - 基础数据图表展示（权限中配置各用户可查阅的数据图表）：0KM不良PPM、售后不良PPM(3mis)、售后不良PPM(12mis)、来料批次合格率、上线不良PPM
    - 动态图表展示：可以通过类型按钮查阅不同类型的指标图表，可以通过时间前后调整按钮查阅不同时间区间指标图表
    - 质量红绿灯：指标达标显示绿色，未达标显示红色。
    - 交互能力：支持下钻，点击指标，可跳转查看当日、当月对应的问题。

### 2.4.3 专项数据分析
- 供应商质量数据分析(来料批次合格率、物料上线不良PPM)：每月分别自动生成Top5供应商清单。
- 制程质量数据分析(制程不合格率)：每月自动形成分类统计，并按类型进行分类月度趋势推移
- 客户质量数据分析(0KM不良PPM、售后不良PPM(3mis)、售后不良PPM(12mis))：每月自动形成分类统计，并按类型进行分类月度趋势推移

### 2.4.4 AI 智能诊断与辅助决策
- 异常自动寻源：
    - 触发逻辑：当监控到某项指标突发飙升时，AI 自动触发分析。
    - 关联分析：AI 自动遍历当天的数据，寻找强相关性因子。
    - *输出示例*：“检测到今日制程不合格率飙升至 5%。关联分析显示：发生批量不良。”
- 自然语言查询：
    - 交互方式：在仪表盘顶部提供一个 AI 对话框。
    - 查询能力：用户可以用自然语言提问，AI 自动转化为 SQL 查询数据库并返回数据详情以及图表。
    - *提问示例*：“帮我把过去三个月MCU产品的 0KM 不良率趋势画成折线图，并对比一下去年同期的数据。”


## 2.5 供应商质量管理

### 2.5.1 IQC 数据集成与监控
- **接口同步策略**：
    - **单向同步**：QMS 通过 API （或者其他途径）定时从 IMS 读取入库检验记录（包含：物料编码、批次号、检验结果 OK/NG、不良描述、不良数量）。
    - **数据用途**：此数据不用于重复显示，而是直接作为 **2.4.1 质量数据中心** 的计算源数据，并最终流向 **2.5.4 绩效评分**。
- **智能触发器**：
    - **NG 自动立案**：监测到 IMS 出现“检验结果 = 不合格”时，自动生成 SCAR 单。
    - **特采标记**：同步 IMS 的特采记录，用于后续追踪该批次物料在产线上的表现。

### 2.5.2 供应商协同与问题闭环
- **交互流程**：
    - **发起端**：IQC 或 产线 发现供应商责任的不良 -> 生成 SCAR 单 -> 邮件通知供应商账号。
    - **供应商端**：供应商登录系统（仅见自己的数据），在线填写 8D 报告（包含围堵措施、根本原因、纠正措施），并必须上传“改善前/后对比图”。
    - **审核端**：SQE 在线审核 8D 报告。系统支持“驳回”功能，驳回时必须填写具体修改意见。
- **AI 辅助审核**：
    - 在 SQE 人工审核前，接入 AI 接口预审 8D 报告。
    - *预审逻辑*：检查是否包含“加强培训”、“加强管理”等空洞词汇，若有，AI 自动提示供应商“措施不具体，请补充作业指导书变更证据”。

### 2.5.3 供应商生命周期与审核
- **准入审核**：新供应商注册时，需在线上传 ISO9001/IATF16949 证书、营业执照等资质文件，系统自动识别有效期并设置“过期预警”。采购工程师/SQE可以对供应商上传资料进行审核以及系统自动识别信息修订，对于供应商提交文件不合格时可以驳回并创建提交任务。
- **供应商变更管理**：供应商在线提交 PCN 申请，采购工程师/SQE 发起内部评估流程，验证通过后系统回传后，供应商执行切换，并上传断点信息。
- **审核管理**：
    - **计划管理**：设定年度审核计划并制定供应商、审核责任SQE以及计划审核月份。
    - **审核周期**：每个月 1 号自动触发审核流程，审核员根据计划审核供应商。
    - **审核指标**：审核员根据 VDA6.3 过程审核 / 体系审核，评估供应商的质量、服务、交付等方面。
    - **审核结果**：审核通过后，系统自动更新供应商状态为“正常”，并记录审核日期。审核未通过，系统自动更新供应商状态为“异常”，并记录审核日期。
    - **问题点追踪**：审核员录入的不符合项 (NC)，自动转化为“任务清单”指派给供应商，直至整改完成关闭。

### 2.5.5 供应商质量目标发布
*（核心逻辑：确立考核基准，支持批量与个别差异化配置，需供应商在线确认签署）*
- **目标制定**：
    - **批量设定**：
        - 支持 SQE 按“物料类别”或“供应商等级”批量设置通用目标。
        - *场景*：设定所有“C级供应商”的来料合格率目标为 99.8%。
    - **单独设定**：
        - 支持针对特定供应商（Supplier ID）进行差异化目标配置。
        - *场景*：针对“核心战略供应商”或“重点整改对象”，单独设定更严格或阶段性的 PPM 目标。
    - **优先级逻辑**：
        - 系统自动遵循 **“单独设定 > 批量设定 > 全局默认值”** 的优先级原则。
        - *计算逻辑*：在计算绩效时，系统优先查询该供应商是否存在有效的“单独目标书”；若无，再回退查询其所属类别的“批量目标书”。
- **辅助决策数据**：
    - 在设定界面，系统自动左右分栏展示：
        - 左侧：拟设定的 2026 年目标值。
        - 右侧：该供应商 2025 年实际达成值（平均值及波动范围），辅助 SQE 评估目标的合理性与挑战性。
- **审批与发布**：
    - **审批流**：SQE 提交 -> 质量经理审核（若目标值低于历史实际值，系统自动高亮提示“目标倒退风险”）-> 发布。
- **供应商签署**：
    - **待办任务**：目标书发布后，供应商门户收到“待签署质量目标”任务。
    - **在线确认**：供应商查阅目标值，点击“确认/签署”。
        - *互锁机制*：若在规定时间（如 1 月 31 日）前未签署，系统限制其查看绩效申诉权限。
    - **版本生效**：系统记录生效日期。绩效计算时，严格依据**数据产生当月**所生效的目标版本进行考核。

### 2.5.6 供应商绩效评价
*（核心逻辑：采用 60 分制扣分后按100分满分折算百分制模型。每月 1 日自动计算，对比 **2.5.5 中签署的目标值**）*
- **基础配置**：
    - **初始分**：每月默认 **60 分**。
    - **目标基准**：**直接调用 2.5.5 模块中该供应商当前生效的目标值**。
- **扣分规则**：
    1.  **来料质量扣分**：
        - *数据源*：2.4.1 计算的“来料检验质量合格率”。
        - *算法*：计算（2.5.5 设定目标值 - 实际值）的差距。
            - 达到目标：扣 0 分。
            - 差距 < 10% 目标值：扣 5 分。
            - 10% ≤ 差距 < 20% 目标值：扣 15 分。
            - 差距 ≥ 20% 目标值：扣 30 分。
    2.  **制程质量扣分**：
        - *数据源*：2.4.1 计算的“物料上线不良 PPM”。
        - *算法*：计算（实际 PPM - 2.5.5 设定目标 PPM）的超标比例。
            - 达到目标：扣 0 分。
            - 0 < 超标 ≤ 50%：扣 5 分。
            - 50% < 超标 ≤ 100%：扣 15 分。
            - 超标 > 100%：扣 30 分。
    3.  **配合度扣分**：
        - *数据源*：SQE 人工评价 + 系统自动规则。
        - *标准*：高（扣 0 分）、中（扣 5 分）、低（扣 10 分）。
    4.  **0公里/售后质量扣分**：
        - *数据源*：2.7 模块中的客诉记录（带严重度标签）。
        - *算法*：
            - 个例物料问题：每次扣 10 分。
            - 批量重大质量异常：每次扣 20 分。
            - 安全问题（如抛锚）：每次扣 30 分。
- **人工评价校核**：
    - SQE在每个月的最后一天，必须在门户填写“配合度评价”（高/中/低）。
    - 系统会根据 SQE 输入的配合度等级，自动计算“配合度扣分”。
    - 针对系统生成的扣分项以及数据，SQE 可在门户查看并进行复核，针对扣减内容可以进行核减并附上详细说明。
- **等级评定**：
*60 分制扣分后按100分满分折算百分制模型*
    - **A级**：得分 > 95。
    - **B级**：80 ≤ 得分 ≤ 95。
    - **C级**：70 ≤ 得分 < 80。
    - **D级**：得分 < 70。
    - *输出*：SQE人工校核后，系统自动生成月度绩效卡。

### 2.5.7 供应商会议与改进监控
*（基于绩效等级的强制干预机制）*
- **会议触发机制**：
    - **自动立项**：当月度绩效生成为 **C级** 或 **D级** 时，自动生成“供应商月度品质改善会议”任务。
    - **参会人员要求**：
        - C级：通知要求供应商**副总级**参加。
        - D级：通知要求供应商**总经理**参加。
- **会议执行与纪律**：
    - **资料上传**：供应商需在门户上传《物料品质问题改善报告》，逾期触发预警。
    - **考勤与纪要**：SQE 录入“实际参会最高级别人员”和“会议纪要”。
    - **违规处罚**：
        - 若标记为“供应商未参会”或“未提交报告”，系统自动在**下个月度**的 2.5.6 绩效评价中，锁定“配合度扣分”项，强制执行额外扣分。

### 2.5.8 PPAP 管理
- **提交管理**：
    - **触发场景**：新物料导入、变更 (ECN)、模具转移或停产一年后恢复生产。
    - **提交等级**：支持配置 Level 1 - Level 5 提交要求（默认为 Level 3）。
    - **文件清单**：系统内置标准 18 项文件检查表（如：PSW、全尺寸测量报告、控制计划、PFMEA、材质证明等）。由SQE指定各项文件的要求上传日期，供应商需在线逐项上传附件。
- **审核与批准**：
    - **在线签核**：SQE 对文件包进行审核，支持“单项驳回”或“整体批准”。
- **年度再鉴定**：
    - 系统自动监控 PPAP 批准日期，满 1 年自动生成“年度产品审核”任务，提醒供应商提交最新报告。

## 2.6 过程质量管理
*（核心逻辑：聚焦于制程不合格数据的采集、责任分类统计以及异常发单闭环。首件/巡检作为线下控制手段，数据暂不接入本模块指标体系。）*

### 2.6.1 生产数据集成
*（为核心指标计算提供准确的“分母”数据）*
- **接口同步策略**：
    - **接口逻辑**：QMS 定时（如每日/每班次）从 IMS 拉取生产入库统计数据。
    - **核心字段**：成品入库数，用于计算“制程不合格率”。
    - **维度要求**：数据需携带“产线”、“工单号”、“日期”标签，以支持多维度的钻取分析。

### 2.6.2 不合格品数据录入与分类
*（核心目的：支持指标“按责任类别进行区分统计”）*
- **数据来源**：
    - **IMS 同步**：同步各工序自动判定的不良记录。
    - **人工补录**：产线/维修工位手动录入未被设备捕捉的外观/组装不良。
- **关键字段定义**：
    - **不良现象**：关联预设的失效类型。
    - **责任类别**：
        - 下拉选项：物料不良、作业不良、设备不良、工艺不良、设计不良。
        - *逻辑关联*：若选择“物料不良”，系统自动关联 2.4.1 的“物料上线不良 PPM”指标。

### 2.6.3 制程质量问题发单管理
- **触发机制**：
    - **自动触发**：预留自动触发配置接口，依据系统实际运行，在未来合适时候进行构建。
    - **手动发起**：PQE 在查看 2.4.3 报表或是不合格品数据清单，可选中关注的数据点、问题点，一键发起整改。
- **闭环流程**：
    - **指派**：根据责任归属，由PQE指派给责任板块担当。
    - **执行**：责任板块担当填写原因分析，制定围堵及长期对策，并上传改善证据。
    - **验证**：对策导入后设定验证期（如：后续 1 周内 2.4 模块显示的指标无超标或是不合格品无再发），由PQE判断验证通过后系统允许关闭。

## 2.7 客户质量管理
*（核心逻辑：实现全渠道客诉的闭环管理，并依据出货数据自动计算 0KM 及 MIS 售后指标，联动财务索赔及供应商扣分。）*

### 2.7.1 出货数据集成
*（为核心指标计算提供准确的“分母”数据）*
- **接口同步策略**：

    - **核心字段**：
        - **当期成品出库量**：用于计算 0KM PPM。
        - **历史滚动出货量**：系统需存储并维护过去 24 个月的分月出货数据表，用于计算 3MIS / 12MIS 等滞后性指标。
    - **维度要求**：数据需携带“客户代码”、“产品类型”、“出货日期”标签。

### 2.7.2 客诉录入与分级受理
*（依据质量问题等级划分标准进行录入与分流）*
- **录入分类**：
    - **0KM 客诉**：客户产线端发现的问题。
    - **售后客诉**：终端市场反馈（需录入：失效里程、购车日期、VIN码）。
- **缺陷等级界定**：
    *系统需内置分级逻辑，录入时必选，直接决定后续的升级汇报机制及供应商扣分权重：*
    - **具体分级方案TBD**
- **一次因解析**：
    - **职责锁定**：依据公司规定，D0-D3 阶段由 CQE 负责。CQE 需在系统内完成“一次解析”，确定责任部门后，任务流转至责任板块进行 D4-D8 处理。
- **自动追溯**：通过录入的追溯信息，联动IMS系统追溯查询系统，自动调取过程记录是否存在异常，如涉及物料失效可通过输入物料编码实现自动追溯查询。

### 2.7.3 8D 闭环与时效管理 (8D Workflow & SLA)
*（遵循时效要求，并实行分级审核）*
- **SLA 动态时效监控**：
    - **8D 报告提交**：7 个工作日内。
    - **报告归档**：10 个工作日内。
    *注：若系统扩展至内部或供应商模块，需支持配置不同的 SLA（如内部制造不良为 3 天回对策）。*
- **8D 执行标准步骤**：
    - **D0-D3 (CQE 完成)**：
        - 填写问题描述 (5W2H) 。
        - 制定围堵措施（需覆盖：在途品、库存品、客户端库存）。
    - **D4-D7 (责任板块担当完成)**：
        - **根本原因**：提供5Why、鱼骨图、FTA、流程分析等工具模板，供责任板块进行分析。
        - **对策验证**：D6 环节必须上传“验证报告”或“测试数据”作为附件，否则无法进入下一步。
        - **标准化**：D7 环节需勾选“是否涉及文件修改”（如 PFMEA/CP/SOP），若选是，需上传修改附件。
    - **D8 水平展开&经验教训**：
        - **水平展开**：需在系统中搜索并关联“类似产品/类似工艺”，将对策推送到相关项目组。
        - **经验教训沉淀**：提交结案时通过勾选是否沉淀经验教训，系统自动弹出《经验教训总结表》弹窗，提交后由责任部门部长审批。
- **审批与归档**：
    - **分级审批流**：
        - C级问题：科室经理审批。
        - A/B级问题：质量部长 + 责任部门部长联合审批。
    - **归档互锁**：
        - 归档时自动触发《8D报告归档检查表》核对流程。

### 2.7.4 索赔管理
- **客户索赔处理**：
    - **录入**：登记客户发来的扣款金额及明细。
    - **手动关联**：可通过筛选将客户索赔与问题清单进行多选关联。

## 2.8 新品质量管理

### 2.8.3 试产目标与实绩管理
*（核心功能：生成试产总结数据的“一张表”）*
- **目标设定**：
    - **关联工单**：在试产开始后，录入本次试产对应的 **IMS 工单号**。
    - **设定指标**：为该批次设定具体的质量目标（如：直通率 > 95%，CPK > 1.33，尺寸合格率 100%）。
- **实绩录入与集成**：
    - **自动抓取**：
        - 系统根据关联的“IMS 工单号”，自动调取该工单的 **投入数、产出数、一次合格数、不良数**，自动计算试产合格率、直通率。
    - **手动补录**：
        - 针对 IMS 无法采集的数据（如：CPK 测算值、破坏性实验结果、外观评审得分），提供在线表格供 QE 手动录入实绩。
- **达成分析报告**：
    - **红绿灯对比**：自动对比“目标值”与“实绩值”。
        - 达标：显示绿色。
        - 未达标：显示红色。
    - **输出**：一键输出《试产质量总结报告 (Excel/PDF)》。

## 2.9 审核管理
*（核心逻辑：实现多体系审核计划的统筹管理，推行数字化检查表以替代纸质记录，并强制追踪不符合项（NC）的整改闭环。）*

### 2.9.1 审核计划与排程
- **多维度计划管理**：
    - **年度计划视图**：支持导入或创建年度审核计划，涵盖以下四类：
        1.  **体系审核** (IATF16949)。
        2.  **过程审核** (VDA 6.3)。
        3.  **产品审核** (VDA 6.5 / 周期性全尺寸)。
        4.  **二方审核** (客户审核 / 潜在客户拜访)。
- **智能提醒**：
    - 系统根据计划日期，提前 N 天（如 7天）邮件通知审核组长及迎审负责人准备资料。
    - **自动延期管理**：若计划未按期执行，需在系统提交“延期申请”并注明理由，经质量部长批准后方可修改计划时间。

### 2.9.2 审核实施与数字化检查表
- **模板库管理**：
    - **内置标准**：系统预置 VDA 6.3 (P2-P7)、VDA 6.5、IATF16949 检查条款库。
    - **自定义模板**：支持创建“专项审核模板”（如：防静电专项审核、异物管理专项审核）。
- **在线打分**：
    - **移动端支持**：可实现审核员携带手机/平板/笔记本，现场逐条录入检查结果（OK/NG）。
    - **证据留存**：对发现的问题点，支持**现场拍照上传**，直接挂载到对应条款下。
    - **自动评分**：
        - 根据 VDA 6.3 规则（如：单项 0 分降级规则），系统自动计算最终得分（如 87%）及等级（A/B/C）。
        - 自动生成《审核报告 (PDF)》，包含得分雷达图和问题清单。

### 2.9.3 问题整改与闭环
- **问题立项**：
    - **自动转化**：审核结束提交后，所有判定为 NG 的条款自动生成“不符合项 (NC) 待办任务”。
    - **任务指派**：由审核组长将 NC 指派给责任板块。
- **整改流程**：
    - **原因与措施**：责任人填写根本原因及纠正措施。
    - **举证关闭**：上传整改后的照片或文件（如：更新后的作业指导书）。
    - **验证**：审核员在线查看证据或进行现场复核，验证有效后关闭 NC。
- **逾期升级**：
    - 设定整改期限（如：一般项 30 天，严重项 7 天）。逾期未关闭的 NC，实现自动跟催并升级抄送。

### 2.9.4 二方审核特别管理
*（针对客户来厂审核的高关注度场景）*
- **迎审档案**：
    - 建立“客户审核台账”，记录：客户名称、审核类型、审核日期、最终得分/结果。
- **问题跟进**：
    - **外部格式兼容**：支持以附件形式上传客户指定的“问题整改清单 (Excel)”。
    - **内部闭环**：依据客户指摘问题清单在系统内部仍需创建对应的问题任务条目，确保不遗漏任何一个客户提出的整改项。
