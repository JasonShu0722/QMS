# Quality Dashboard Implementation

## Overview
This document describes the implementation of the Quality Data Dashboard (质量数据仪表盘) frontend components for the QMS system.

## Components Implemented

### 1. QualityDashboard.vue
**Location:** `frontend/src/views/QualityDashboard.vue`

**Features:**
- **Metric Cards Grid**: Displays quality metrics with color-coded status indicators (green for good, red for danger, yellow for warning)
- **Trend Charts**: Interactive ECharts visualizations showing metric trends over time
- **Dynamic Controls**: Date picker, metric type selector, and date range selector for filtering data
- **Drill-down Functionality**: Click on metric cards to view detailed trends
- **Specialized Analysis Tabs**:
  - Supplier Quality Analysis (Top 5 suppliers)
  - Process Quality Analysis (defect rates and FPY trends)
  - Customer Quality Analysis (0KM, 3MIS, 12MIS PPM trends)
- **Responsive Design**: Mobile-friendly layout with Tailwind CSS
- **Real-time Data Refresh**: Manual refresh button to reload all data

**Key Interactions:**
- Click metric cards to drill down into specific metrics
- Use quick date range buttons (7 days, 30 days, 3 months)
- Switch between analysis tabs to view different perspectives
- All charts are responsive and resize automatically

### 2. AIAssistant.vue
**Location:** `frontend/src/components/AIAssistant.vue`

**Features:**
- **Floating Action Button**: Fixed position button in bottom-right corner
- **Chat Interface**: Dialog-based chat interface with message history
- **Natural Language Queries**: Users can ask questions in plain Chinese
- **Data Visualization**: Automatically renders tables and charts from AI responses
- **ECharts Integration**: Displays charts generated by AI based on query results
- **Message History**: Maintains conversation context
- **Loading States**: Shows loading indicator while AI processes queries
- **Mobile Responsive**: Adapts to mobile screen sizes

**Example Queries:**
- "查询上个月来料批次合格率最低的5个供应商"
- "帮我把过去三个月MCU产品的0KM不良率趋势画成折线图"
- "对比今年和去年同期的制程不合格率"

## API Integration

### Quality Metrics API Client
**Location:** `frontend/src/api/quality-metrics.ts`

**Endpoints:**
- `getDashboard(targetDate?)` - Get dashboard summary data
- `getMetricTrend(params)` - Get metric trend data
- `getTopSuppliers(params)` - Get top 5 suppliers by metric
- `getProcessAnalysis(params)` - Get process quality analysis
- `getCustomerAnalysis(params)` - Get customer quality analysis
- `diagnoseAnomaly(data)` - AI anomaly diagnosis
- `naturalLanguageQuery(data)` - AI natural language query

## Type Definitions

### Quality Metrics Types
**Location:** `frontend/src/types/quality-metrics.ts`

**Key Types:**
- `MetricType` - Enum of all metric types
- `MetricStatus` - 'good' | 'warning' | 'danger' | 'unknown'
- `TrendDirection` - 'up' | 'down' | 'stable'
- `DashboardResponse` - Dashboard data structure
- `MetricTrendResponse` - Trend data structure
- `TopSuppliersResponse` - Top suppliers data structure
- `ProcessAnalysisResponse` - Process analysis data structure
- `CustomerAnalysisResponse` - Customer analysis data structure
- `AIDiagnoseRequest/Response` - AI diagnosis data structures
- `AIQueryRequest/Response` - AI query data structures

## Chart Visualizations

### ECharts Integration
All charts use ECharts 5.6.0 with the following configurations:

1. **Trend Line Chart**
   - Shows actual values vs target values
   - Smooth curves with gradient area fill
   - Interactive tooltips with cross-axis pointer
   - Responsive to window resize

2. **Top Suppliers Bar Chart**
   - Color-coded bars based on status (green/yellow/red)
   - Labels showing exact values
   - Horizontal category axis with supplier names

3. **Process Quality Line Chart**
   - Dual Y-axis for defect rate and FPY
   - Monthly trend data
   - Legend for multiple series

4. **Customer Quality Multi-line Chart**
   - Three series: 0KM PPM, 3MIS PPM, 12MIS PPM
   - Monthly trend data
   - Color-coded lines for easy distinction

## Notes

### Performance Considerations
- Charts are initialized once and reused (not recreated on data updates)
- Proper cleanup in `onUnmounted` to prevent memory leaks
- Debounced resize handlers for chart responsiveness
- Lazy loading of chart data (only load when tab is active)

### Accessibility
- Semantic HTML structure
- ARIA labels on interactive elements
- Keyboard navigation support (via Element Plus)
- Color contrast meets WCAG AA standards

### Internationalization
- All UI text is in Simplified Chinese
- Date formats use ISO 8601 (YYYY-MM-DD)
- Number formats use Chinese conventions
