# QMS 项目功能需求条目整理

## 1. 整理依据

本次需求梳理基于以下三类信息统一整理：

- `/.kiro/steering/product.md`
- `/.kiro/steering/structure.md`
- 当前对外汇报使用的模块优先级梳理图

整理目标不是简单复述模块名称，而是形成一份便于后续持续维护的需求总表，满足三件事：

- 明确每个模块下的功能条目与期望能力
- 明确整体优先级、范围和阶段定位
- 为后续联调验证后的状态维护提供唯一数据入口

## 2. 需求分层口径

### 2.1 优先级

- `高`
  直接影响 QMS 主链路价值交付，应优先联调、验证、闭环。
- `中`
  对管理效率、数据分析、运营抓手有明显帮助，建议跟随一期主线推进。
- `低/预留`
  作为后续扩展、接口预留或二期延展能力。

### 2.2 范围

- `核心需求`
  当前项目应明确纳入交付范围的能力。
- `期望增强`
  已被业务明确提出、建议纳入后续增强排期的能力。
- `预留功能`
  当前先做结构预留，不要求一期落地。

### 2.3 状态

- `待开发`
- `开发中`
- `已开发待验证`
- `已验证完成`
- `预留`

## 3. 模块化需求清单

### 3.1 系统基础与门户管理

重点承担整个平台基础底座能力：

- 权限配置矩阵
- 内外部统一登录与门户切换
- 动态任务工作台
- 消息通知中心
- 操作日志与审计追踪
- 用户注册审批与账号冻结
- 个人中心与电子签名
- 公告管理与强制阅读
- 移动端/PDA 适配
- 功能开关与灰度发布

### 3.2 质量数据总览与仪表盘

重点承担数据集成、趋势分析与管理可视化：

- 指标自动统计与源数据集成
- 可视化看板
- 趋势分析与异常预警

- 专项质量分析

### 3.3 物料质量管理

- 供方绩效评价
- 供应商审核进度管理
- PPAP 认可归档
- 供应商变更管理
- 条码防错与批次校核

### 3.4 过程质量管理

重点承担制程质量记录与问题闭环：

- 生产数据集成
- 不良记录
- 问题发单及整改跟踪
- 数据联动 AI 诊断验证
- 出货报告归档
- 制程良率与直通率跟踪

### 3.5 客户质量管理

重点承担客诉、索赔和客户质量指标闭环：

- 出货数据集成

- 数据联动 AI 诊断验证

### 3.6 新品质量管理

重点承担经验导入、试产跟进与新品问题前置管理：

- 阶段交付物评审

- 量产移交归档
- 试产总结与目标达成分析

### 3.7 审核管理

重点承担多体系审核、数字化检查表与 NC 闭环：

- 审核计划
- 问题整改关闭验证
- 进展晾晒
- 数字化审核表
- 资料归档
- 审核模板库
- 客户/二方审核台账

### 3.8 预留与期望功能

重点承担中长期扩展：

- 质量成本统计
- 仪器计量管理
- 试验管理
- 版本与双环境发布管理
- 灰度白名单控制

## 4. 可视化面板文件

本次新增两个配套文件：

- [qms-requirements-data.js](/E:/WorkSpace/QMS/doc/requirements-panel/qms-requirements-data.js)
- [qms-requirements-panel.html](/E:/WorkSpace/QMS/doc/requirements-panel/qms-requirements-panel.html)

其中：

- `qms-requirements-data.js`
  作为唯一维护数据源，记录模块、功能条目、优先级、范围、阶段、状态和验收口径。
- `qms-requirements-panel.html`
  作为静态可视化面板，自动读取上面的数据源，并按“功能模块”和“整体优先级”两种维度展示。
  同时支持在面板中直接切换功能状态，并通过浏览器本地存储保留这些状态覆盖。

## 5. 后续维护方式

后续每当一个功能完成“联调 + 验证确认”时，建议优先按下面方式维护：

1. 打开 [qms-requirements-panel.html](/E:/WorkSpace/QMS/doc/requirements-panel/qms-requirements-panel.html)
2. 在“需求明细台账”中找到对应条目
3. 直接切换状态下拉框
4. 面板会即时刷新模块进度、优先级看板与汇总数据

如果需要沉淀这次状态结果，可以点击面板中的“导出状态快照”。

如果后续希望把当前状态作为新的默认值固化到仓库，再执行下面这套操作：

1. 打开 [qms-requirements-data.js](/E:/WorkSpace/QMS/doc/requirements-panel/qms-requirements-data.js)
2. 找到对应功能条目
3. 更新 `status`
4. 如有必要同步补充 `acceptance`
5. 保存后重新打开 [qms-requirements-panel.html](/E:/WorkSpace/QMS/doc/requirements-panel/qms-requirements-panel.html) 查看结果

建议的状态流转：

- `待开发` → `开发中`
- `开发中` → `已开发待验证`
- `已开发待验证` → `已验证完成`

如果后续你希望把这个面板进一步升级成：

- 自动读取数据库或接口状态
- 支持导出汇报版图表
- 支持按负责人/版本/迭代过滤

可以在这套数据结构的基础上继续扩展，不需要推翻重做。
